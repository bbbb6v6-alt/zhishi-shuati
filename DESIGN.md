# 党史刷题软件设计方案

## 产品定位
专业党史学习工具，帮助用户高效记忆和理解党史知识。

## 设计风格
**中国红主题 + 卡片式布局** — 庄重大气又不失学习工具的亲和力

### 视觉关键词
- 庄重、权威、正式（符合党史学习场景）
- 清晰、高效、专注（学习工具核心属性）

### 配色方案
| 用途 | 色值 | 说明 |
|------|------|------|
| Primary (主色) | #C41E3A | 中国红，体现党建元素 |
| Background | #FEF2F2 | 淡红底色，温暖护眼 |
| Surface (卡片) | #FFFFFF | 纯白卡片，高对比清晰 |
| Text Primary | #1F2937 | 深灰文字，易读 |
| Text Secondary | #6B7280 | 辅助文字 |
| Correct (正确) | #059669 | 翠绿，正确答案 |
| Wrong (错误) | #DC2626 | 红色，错误答案 |
| Warning | #D97706 | 橙色，警示/重点 |

### 字体排版
- 标题：Bold，28-32px
- 正文：Regular，16-18px
- 辅助：Regular，13-14px
- 行高：1.5-1.6 倍

### 动效设计
- 答题卡片切换：300ms ease-out 滑入
- 答案揭示：fade-in 200ms
- 按钮反馈：scale 0.98 按压效果

## 页面结构

### 1. 首页 (Index)
- 顶部：标题栏「党史刷题」
- 中部：学习统计卡片（正确率、总题数、错题数）
- 下部：刷题入口按钮组
  - 顺序练习（按题库顺序）
  - 随机练习（打乱顺序）
  - 错题复习（专项训练）
  - 题型专练（判断/选择/简答）

### 2. 刷题页 (Practice)
- 顶部：进度条 + 题号
- 中部：题目卡片
  - 题型标签（判断/单选/多选/简答）
  - 题干文字
  - 选项列表（判断题为对/错二选一）
  - 简答题为文本输入框
- 底部：提交按钮

### 3. 答题结果页 (Result)
- 顶部：正确/错误状态标识
- 题目回显：原题 + 用户答案 + 正确答案
- 解析区域：高亮显示关键知识点
- 底部：下一题 / 返回首页

### 4. 错题集页 (WrongQuestions)
- 顶部：错题统计
- 列表：错题卡片（题目 + 错误原因提示）
- 支持重新练习错题

## 交互设计

### 答题流程
1. 选择刷题模式 → 进入刷题页
2. 阅读题目 → 选择/填写答案
3. 点击提交 → 显示结果和解析
4. 点击下一题 → 继续答题
5. 全部完成 → 显示总结统计

### 数据存储
- 题库：服务端数据库
- 用户答题记录：本地 AsyncStorage（设备级）
- 错题记录：本地 + 服务端同步

## 技术实现

### 前端 (Expo + React Native)
- 路由：Stack 导航
- 状态：React Context 管理答题状态
- 存储：AsyncStorage 本地持久化

### 后端 (Express)
- 题库管理 API
- 答题记录 API
- 错题统计 API
